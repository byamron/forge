# Artifact Templates

Reference for the artifact-generator agent. Contains exact templates and formatting rules for each Claude Code artifact type.

## CLAUDE.md Entry

**Format:** Imperative statement, 1-2 lines. No headers, no verbose explanation.

```markdown
Use pnpm for all package operations, never npm or yarn.

Always use TypeScript strict mode. Enable `strict: true` in tsconfig.json.
```

**Validation:**
- Max 2 lines per entry
- Must be actionable (imperative voice)
- No markdown headers within entries
- Total CLAUDE.md should stay under 200 lines. Files over 200 lines consume more context and may reduce adherence. Upper bound ~500 lines — beyond that, move reference content to skills

## Rule (`.claude/rules/<name>.md`)

**Filename:** kebab-case, descriptive (e.g., `testing-conventions.md`, `api-design.md`, `react-components.md`)

**Template:**
```markdown
---
paths:
  - "**/*.test.ts"
---

Use vitest for all test files.
Use `describe`/`it` block structure.
Prefer `toEqual` over `toBe` for object comparisons.
Mock external services, never mock the database.
```

**Validation:**
- `paths` frontmatter (plural, YAML list) is optional but recommended for scoping
- Rules with `paths` load only when Claude reads matching files. Rules without `paths` load at session start
- Paths use glob patterns (e.g., `**/*.tsx`, `src/api/**/*.ts`)
- One topic per file
- Filename matches content topic
- Content is concise — extract detailed guidance to reference docs

## Hook (JSON in settings.json)

**PostToolUse auto-format (Prettier):**
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "npx prettier --write \"$CLAUDE_TOOL_INPUT_FILE_PATH\"",
            "timeout": 10
          }
        ]
      }
    ]
  }
}
```

**PostToolUse auto-lint (ESLint):**
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "npx eslint --fix \"$CLAUDE_TOOL_INPUT_FILE_PATH\"",
            "timeout": 15
          }
        ]
      }
    ]
  }
}
```

**PostToolUse auto-format (Python/Black):**
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "black \"$CLAUDE_TOOL_INPUT_FILE_PATH\"",
            "timeout": 10
          }
        ]
      }
    ]
  }
}
```

**PostToolUse auto-format (Rust/rustfmt):**
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "rustfmt \"$CLAUDE_TOOL_INPUT_FILE_PATH\"",
            "timeout": 10
          }
        ]
      }
    ]
  }
}
```

**Validation:**
- Valid JSON
- Matcher patterns: case-sensitive, no spaces around `|`
- Handler type must be one of: `command`, `http`, `prompt`, `agent`
- Timeout in seconds. Defaults: command=600, http=30, prompt=30, agent=60
- Commands should use `$CLAUDE_TOOL_INPUT_FILE_PATH` for file-specific hooks

## Skill (`.claude/skills/<name>/SKILL.md`)

**Directory structure:**
```
.claude/skills/<name>/
├── SKILL.md
├── scripts/       (optional — for deterministic steps)
└── references/    (optional — for deep documentation)
```

**SKILL.md template:**
```yaml
---
name: skill-name-in-kebab-case
description: >
  Brief description of what this skill does. Include trigger phrases
  that users might say. Use when you want to [specific use case].
  Also triggered by [alternative phrasings].
---
```

```markdown
<!-- Draft generated by Forge — test and iterate -->

You are running the [skill name] workflow. Follow these steps:

## Step 1: [First action]
[Clear imperative instructions]

## Step 2: [Second action]
[Clear imperative instructions]

## Step 3: [Final action]
[Clear imperative instructions]
```

**Optional frontmatter fields:**
- `argument-hint`: Hint text shown in slash command autocomplete
- `user-invocable`: Whether the skill appears in slash command menu (default true)
- `disable-model-invocation`: If true, skill is invisible to auto-triggering
- `allowed-tools`: Tools the skill can use
- `model`: Model override (`sonnet`, `opus`, `haiku`)
- `effort`: Reasoning effort level (`low`, `medium`, `high`, `max`)
- `context`: Additional context files to load
- `agent`: Run as a subagent with these settings
- `paths`: File patterns that trigger this skill

**Validation:**
- `name` must be kebab-case, max 64 characters
- `description` must be <1024 characters
- No XML tags in frontmatter values
- Description should include "Use when..." language
- Body uses imperative voice with clear step structure

## Agent (`.claude/agents/<name>.md`)

**Template:**
```yaml
---
name: agent-name
description: >
  What this agent does and when it should be used.
model: sonnet
effort: low
maxTurns: 5
disallowedTools:
  - Write
  - Edit
---
```

```markdown
<!-- Draft generated by Forge — test and iterate -->

You are the [agent name]. Your role is to [primary responsibility].

## Responsibilities
- [Responsibility 1]
- [Responsibility 2]

## Constraints
- [Constraint 1]
- [Constraint 2]

## Output
[What the agent should produce]
```

**Optional frontmatter fields:**
- `tools`: Specific tools the agent can use (inherits all if omitted)
- `permissionMode`: `default`, `acceptEdits`, `dontAsk`, `bypassPermissions`, `plan`
- `skills`: Skill content injected at startup
- `mcpServers`: MCP servers available to the subagent
- `hooks`: Lifecycle hooks scoped to the subagent
- `memory`: `user`, `project`, or `local`
- `background`: Run as background task
- `isolation`: `worktree` for git worktree isolation
- `initialPrompt`: Auto-submitted first user turn when agent runs as main

**Validation:**
- `name`: lowercase letters and hyphens
- `model`: `sonnet`, `opus`, `haiku`, full model ID, or `inherit` (default: `inherit`)
- `effort`: `low`, `medium`, `high`, or `max` (max is Opus only)
- `maxTurns`: integer, keep low (3-10) to control cost
- `disallowedTools`: list tools the agent should not use

## Reference Document (`.claude/references/<name>.md`)

**Template:**
```markdown
# [Topic Name]

## Overview
[Brief context]

## [Section 1]
[Detailed guidance with examples]

## [Section 2]
[Detailed guidance with examples]

## Examples
[Practical examples showing correct usage]
```

**Linking from CLAUDE.md or rules:**
```markdown
For detailed API design conventions, see `.claude/references/api-design.md`.
```

**Validation:**
- Use clear markdown headings
- Include practical examples
- Keep focused on one topic
- Must be linked from CLAUDE.md or a rule to be discoverable
