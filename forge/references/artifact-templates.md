# Artifact Templates

Reference for generating Claude Code artifacts. Follow these templates and validation rules.

## CLAUDE.md Entry

Imperative statement, 1-2 lines. No headers.
```markdown
Use pnpm for all package operations, never npm or yarn.
```
- Max 2 lines per entry, actionable imperative voice
- Total CLAUDE.md should stay under 200 lines (upper bound ~500 — beyond that, extract to skills)

## Rule (`.claude/rules/<name>.md`)

Filename: kebab-case (e.g., `testing-conventions.md`). One topic per file.
```markdown
---
paths:
  - "**/*.test.ts"
---

Use vitest for all test files.
Prefer `toEqual` over `toBe` for object comparisons.
```
- `paths` (plural, YAML list) is optional — rules with `paths` load only when matching files are read
- Content is concise, imperative voice

## Hook (JSON in `.claude/settings.json`)

All hooks follow this structure — substitute the tool command:
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "<tool> \"$CLAUDE_TOOL_INPUT_FILE_PATH\"",
            "timeout": 10
          }
        ]
      }
    ]
  }
}
```
Common commands: `npx prettier --write`, `npx eslint --fix`, `black`, `rustfmt`, `cargo fmt`

- Matcher patterns: case-sensitive, no spaces around `|`
- Handler types: `command`, `http`, `prompt`, `agent`
- Timeout defaults: command=600s, http=30s, prompt=30s, agent=60s
- When merging into existing settings.json, preserve all existing hooks

## Skill (`.claude/skills/<name>/SKILL.md`)

```yaml
---
name: skill-name-in-kebab-case
description: >
  What this skill does. Use when you want to [specific use case].
---
```
```markdown
<!-- Draft generated by Forge — test and iterate -->

You are running the [skill name] workflow. Follow these steps:

## Step 1: [Action]
[Imperative instructions]
```
- `name`: kebab-case, max 64 chars. `description`: <1024 chars, include "Use when..." language
- Optional fields: `argument-hint`, `user-invocable`, `allowed-tools`, `model`, `effort`, `context`, `paths`

## Agent (`.claude/agents/<name>.md`)

```yaml
---
name: agent-name
description: >
  What this agent does and when it should be used.
model: sonnet
effort: low
maxTurns: 5
disallowedTools:
  - Write
  - Edit
---
```
```markdown
<!-- Draft generated by Forge — test and iterate -->

You are the [agent name]. Your role is to [primary responsibility].

## Constraints
- [Constraint 1]
```
- `model`: `sonnet`/`opus`/`haiku`/`inherit`. `effort`: `low`/`medium`/`high`/`max`
- Optional fields: `tools`, `permissionMode`, `skills`, `mcpServers`, `hooks`, `memory`, `background`, `isolation`

## Reference Document (`.claude/references/<name>.md`)

```markdown
# [Topic Name]

## Overview
[Brief context]

## [Section]
[Detailed guidance with examples]
```
- Must be linked from CLAUDE.md or a rule to be discoverable
- Add pointer: `For detailed X conventions, see .claude/references/Y.md`
